# Паспорт автоматизации бизнеса

Интерактивный веб-сервис для предварительной диагностики ручных и повторяющихся бизнес-процессов.

Сервис помогает предпринимателю, руководителю или эксперту понять:

* какие действия можно автоматизировать;
* сколько времени сотрудники тратят на процесс;
* какой объём времени потенциально можно освободить;
* какой формат цифрового решения может подойти;
* какие данные потребуются для запуска;
* какие риски и ограничения необходимо учесть;
* с чего начать первую рабочую версию проекта.

## Для кого предназначен проект

* малый бизнес;
* предприниматели;
* самозанятые;
* эксперты;
* руководители небольших команд;
* компании с повторяющимися ручными процессами.

## Основные функции

* многошаговая диагностика бизнес-процесса;
* расчёт текущих трудозатрат за месяц;
* предварительная оценка потенциала автоматизации;
* расчёт возможной экономии времени;
* подбор подходящего формата решения;
* формирование ожидаемых эффектов;
* выявление рисков и ограничений;
* создание черновика первой версии решения;
* сохранение заполненных данных в браузере;
* копирование готового паспорта;
* скачивание результата в формате Markdown;
* печать итогового документа;
* переход в Telegram для индивидуального разбора задачи.

## Важный принцип

Клиент не выбирает конкретные технологии и инструменты разработки.

Сервис описывает результат понятным бизнес-языком:

* мини-сервис;
* автоматизация процесса;
* ИИ-ассистент с контролем человека;
* система аналитики и отчётности;
* комплексное решение с интеграциями.

Конкретная технология определяется после изучения реального процесса, данных, ограничений и требований к безопасности.

## Технологии

* HTML5;
* CSS3;
* JavaScript;
* LocalStorage;
* без внешних библиотек;
* без серверной части;
* без платных API.

## Локальный запуск

1. Скачайте или клонируйте репозиторий.
2. Откройте папку проекта.
3. Запустите файл `index.html` в браузере.

Сервис работает без установки дополнительных зависимостей.

## Ограничения

Результат диагностики является предварительной оценкой.

Он не является:

* окончательным техническим заданием;
* коммерческим предложением;
* гарантией экономии времени;
* гарантией финансового результата;
* подтверждением технической возможности конкретной интеграции.

Фактический эффект зависит от качества данных, правил процесса, доступности систем и условий внедрения.
